# Home Screen (Главное меню) — Design

## Контекст

Сейчас при загрузке `index.html` пользователь сразу видит вкладку "Másele"
(решение задач) с полной верхней панелью вкладок (`Másele`, `Kalkulyator`,
`Baza`, `Shınıǵıw`, `Admin`). Отдельного стартового экрана-меню нет.

## Цель

Добавить стартовый экран ("главное меню"), с которого пользователь выбирает
один из 4 основных разделов. После выбора — обычный режим работы (как сейчас,
с панелью вкладок).

## Поведение

### Первый визит (нет `localStorage.lastTab`)
- Показывается главный экран (`home-mode` на `.app-shell`).
- Верхняя панель видна полностью (лого, переключатель темы, вход),
  но **без** ряда вкладок `.tab-nav`.
- На экране — заголовок-приветствие и сетка из 4 карточек:
  - Másele (решить задачу) → `data-tab="solver"`
  - Kalkulyator → `data-tab="calc"`
  - Baza (база знаний) → `data-tab="kb"`
  - Shınıǵıw (тренажер) → `data-tab="tasks"`

Карточки используют существующие SVG-иконки из соответствующих кнопок вкладок.

### Клик по карточке
- `.app-shell` теряет класс `home-mode` (становится виден `.tab-nav`).
- Активируется выбранная вкладка/панель (как при обычном клике по `.tab-btn`).
- В `localStorage.lastTab` сохраняется id выбранной вкладки.

### Повторный визит (есть `localStorage.lastTab`)
- Главный экран **не показывается**.
- Сразу открывается сохранённая вкладка в обычном режиме (`tab-nav` виден).
- Если сохранённое значение невалидно (например, `admin`, а пользователь не
  админ) — fallback на главный экран.

### Возврат на главный экран
- Отдельной кнопки "На главную" нет (вне рамок задачи).
- Чтобы снова увидеть главный экран — нужно очистить `localStorage.lastTab`
  (например, через DevTools) или это будет добавлено отдельной задачей в
  будущем.

## Реализация

### `index.html`
- Новая панель `panel-home` (первая в `.panels`), с классом `panel`.
  Активна по умолчанию вместо `panel-solver`, если `home-mode` активен.
- Внутри — заголовок и `<div class="home-cards">` с 4 `<button class="home-card" data-tab="...">`,
  переиспользующими SVG-иконки из `.tab-nav`.
- На `<div class="app-shell">` по умолчанию добавляется класс `home-mode`
  (если JS не отключит его при наличии `localStorage.lastTab` — делается до
  первой отрисовки, синхронным `<script>` в `<head>`, аналогично текущему
  переключению темы, чтобы избежать "мигания").

### `style.css`
- `.app-shell.home-mode .tab-nav { display: none; }`
- `.app-shell:not(.home-mode) #panel-home { display: none; }`
- `.home-cards` — CSS grid (2 колонки на десктопе, 1 на мобильных).
- `.home-card` — карточка с иконкой, заголовком и коротким описанием,
  hover-эффект в стиле существующих `.btn-primary`/`.tab-btn`.

### `app.js`
- При `DOMContentLoaded`:
  - если `home-mode` не был снят синхронным скриптом и `localStorage.lastTab`
    отсутствует/невалиден → главный экран остаётся активным, `panel-home`
    получает класс `active`, остальные панели — без `active`.
  - если `localStorage.lastTab` валиден → убрать `home-mode`, активировать
    соответствующую кнопку `.tab-btn` и панель `panel-<tab>` (повторное
    использование текущей логики переключения вкладок).
- Обработчик клика на `.home-card`:
  - убрать класс `home-mode` у `.app-shell`;
  - программно "кликнуть" по соответствующей `.tab-btn` (или вызвать общую
    функцию переключения вкладок) — переиспользуем существующий код;
  - записать `data-tab` в `localStorage.lastTab`.

## Edge cases
- Пользователь без прав admin: карточка Admin не показывается на главном
  экране (только 4 карточки, как в подтверждённом дизайне). Если
  `localStorage.lastTab === 'admin'`, но роль не admin — fallback на главный
  экран.
- Тема (светлая/тёмная) и переключатель темы работают на главном экране как
  обычно, без изменений в `toggleTheme()`.

## Тестирование
- Вручную: очистить `localStorage`, перезагрузить — должен открыться главный
  экран с 4 карточками.
- Клик по каждой карточке открывает соответствующий раздел и сохраняет выбор.
- Перезагрузка после выбора — открывается сохранённый раздел, без главного
  экрана.
- Очистка `localStorage.lastTab` и перезагрузка — снова главный экран.
